# Neural Launch Positioning And Growth Plan

## Purpose

Turn the current Neural beta into a launchable product story with a practical growth plan that fits the software that actually exists.

## Product Truth

Neural is not yet a broad prediction-market platform.

Neural is:

- a Kalshi-first SDK
- a prediction-market workflow layer
- a beta product with honest boundaries

The supported story is:

- Kalshi auth
- Kalshi market data
- paper trading
- selected live trading workflows
- CLI health and capability checks
- Polymarket reads and streaming only

Everything else should stay experimental, deferred, or premium-roadmap only.

## Core Positioning

### Recommended One-Liner

Kalshi-first SDK for prediction-market data, paper trading, and selected live workflows.

### Recommended Two-Sentence Version

Neural gives developers a narrow, production-minded path into prediction-market trading without starting from raw venue APIs. Start with Kalshi market data, paper trading, and CLI checks today; keep Polymarket in read-and-stream beta while the broader surface hardens.

## ICP

### Primary

- solo quant developers
- small trading teams
- prediction-market power users who want a programmable workflow, not just a venue account

### Secondary

- hedge funds or research pods exploring event-driven prediction-market strategies
- developer tool builders who want a reusable workflow layer for a prediction-market product

## Messaging Pillars

### 1. Honest Beta

The product promise matches the product surface.

### 2. Faster Than Raw APIs

The value is not "we have an API." The value is:

- cleaner startup path
- one workflow surface
- paper trading and operator checks in the same SDK

### 3. Kalshi-First Reliability

Lead with the strongest supported path instead of pretending all venues are equally ready.

### 4. Open-Core Clarity

Be explicit about what is free core, what is experimental, and what is future premium work.

## Launch Assets

### Must Have

- landing page with honest hero and real waitlist CTA
- beta release notes
- beta release checklist
- competitive landscape brief
- developer adoption brief
- one short FAQ
- one simple source-attributed waitlist flow

### Nice To Have

- short product demo gif or terminal capture
- one operator workflow screenshot from the CLI or SDK examples
- one public example strategy walkthrough

## Growth Motion

### Phase 1: Ship And Collect Demand

- publish the SDK beta
- deploy the landing page
- drive all traffic to the waitlist and docs
- optimize for qualified developer signups, not vanity traffic

### Phase 2: Activate Developers

- help new users reach first success quickly:
- install package
- run `neural doctor`
- read one Kalshi market
- run one paper-trading example

### Phase 3: Learn And Tighten

- review weekly waitlist conversion and source data
- review which docs paths and CTAs drive the highest-quality signups
- rewrite onboarding around the strongest activation path

- direct outreach to Kalshi or Polymarket-adjacent builders
- YC / startup / quant communities where prediction markets are already discussed

## Definition Of Done For "Shift To Growth"

Neural can say it is done with core beta development when:

- the landing page is live
- the SDK beta is published or publish-ready
- release notes and checklist are stable
- public messaging matches the actual product
- the low-confidence modules are not in the default story
- the remaining open engineering items are operational hardening rather than product-definition work
