---
title: 'TradingClient API'
description: 'Execute Kalshi trades with the Neural TradingClient facade and optional dependency injection.'
---

`neural.trading.client.TradingClient` is Neural's exchange-facing trading facade. It defaults to Kalshi, preserves a stable public interface, and can also construct the read-only Polymarket US adapter where implemented.

The base SDK is enough to import and inspect the client. Install `neural-sdk[trading]` when you want live Kalshi execution or streaming helpers; keep the Polymarket US path in read/stream beta until live order placement is explicitly promoted.

## Construction

```python
from neural.trading import TradingClient

client = TradingClient(
    api_key_id="...",            # optional override
    private_key_pem=b"...",      # bytes or None to load from env/files
    env="prod",                  # only production is supported
    timeout=15,
)
```

Omitting arguments defers to `neural.auth.env` helpers so environment variables or secrets files drive the configuration.

To construct the Polymarket US adapter instead:

```python
client = TradingClient(
    exchange="polymarket_us",
    polymarket_us_api_key="...",
    polymarket_us_api_secret=b"...",
    polymarket_us_passphrase="...",
)
```

During the current beta, `exchange="polymarket_us"` supports reads and streaming only. Live order placement is not yet part of the supported surface.

## Dependency Injection

`TradingClient` accepts a `client_factory` callable to inject a mock or preconfigured `kalshi-python` client. That is useful for tests and paper simulations.

```python
class FakeClient:
    def __init__(self, **kwargs):
        self.portfolio = ...
        self.markets = ...
        self.exchange = ...

client = TradingClient(client_factory=lambda **kwargs: FakeClient(**kwargs))
```

## Proxies

After initialization, top-level methods expose the stable public trading surface:

- `client.get_positions()`
- `client.list_markets()`
- `client.place_order()`

The client also exposes compatibility proxies at `client.portfolio`, `client.markets`, and `client.exchange` for older integrations.

```python
positions = client.get_positions()
markets = client.list_markets(sport="football", limit=25)
order = client.place_order(
    market_id="KXNFLGAME-25SEP25SEAARI-SEA",
    side="yes",
    quantity=100,
    order_type="limit",
    price=55,
)
```

The compatibility proxies accept `sport`, `category`, `limit`, and `sports_only`. If you need raw Kalshi REST parameters like `series_ticker`, use `KalshiHTTPClient` directly.

## Resource Management

`TradingClient` implements context manager hooks:

```python
with TradingClient() as client:
    ...
```

`close()` gracefully shuts down the underlying client if it exposes a `close()` method.

## Error Messaging

If `kalshi_python` is missing, the client raises a clear `ImportError`:

```
kalshi_python is required for neural.trading. Install with: pip install 'kalshi-python>=2'
```

This keeps import-time errors descriptive and prevents silent failures.

## Best Practices

- Reuse a single `TradingClient` instance per process to benefit from connection pooling.
- Inject custom factories in tests to avoid hitting real endpoints.
- Combine with `OrderManager` to keep strategy logic independent from execution details.
- Call `client.capabilities()` before wiring a new exchange into live execution paths.
- Treat Polymarket US live order placement and FIX-based automation as out of scope for the supported beta until those surfaces are explicitly promoted.

The facade gives you a stable surface even when Kalshi’s SDK evolves, so your algorithms stay focused on trading logic.
