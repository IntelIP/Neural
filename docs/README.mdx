## What is Neural?

Neural is an algorithmic trading SDK for building prediction-market workflows with a Kalshi-first public beta.

The public beta is intentionally narrow. Neural gives you a programmable path for researching markets, rehearsing execution, and automating selected live trading workflows without implying that the broader repository surface is production-ready.

Use it to:

- Develop systematic, data-driven strategies
- Backtest your approaches on historical data
- Execute Kalshi trades programmatically with risk controls
- Rehearse workflows in paper mode before risking capital

## Current beta scope

The supported beta story today is:

- Kalshi auth and market data
- Paper trading
- Selected Kalshi live trading flows through `TradingClient`
- Minimal CLI health checks

Polymarket US currently ships as read/stream-only beta support. Sentiment, deployment, FIX streaming, and broader platform surfaces are still experimental.

Installation follows the same split:

- `pip install neural-sdk` for the base SDK, CLI, and supported beta imports
- `pip install "neural-sdk[trading]"` for Kalshi live trading, streaming, and FIX helpers
- `pip install "neural-sdk[sentiment]"` for research-only sentiment tooling
- `pip install "neural-sdk[deployment]"` for the experimental Docker deployment helpers

Keep the beta boundary narrow in production. Use the base SDK and `trading` extra for the public path; treat sentiment, FIX, and deployment helpers as operator-owned experiments until they are promoted.

## Getting Started

If you're new to Neural, start with the quickstart to validate your environment, pull live Kalshi data, and run the paper-trading workflow end to end.

<Note>
Neural is still in beta. Trade at your own risk, keep experimental modules behind review, and prefer the documented Kalshi-first path until the broader surface is hardened.
</Note>
